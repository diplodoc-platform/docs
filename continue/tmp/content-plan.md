# План обновления страницы "Создание PDF из документации"

**Цель:** Превратить набор тезисов в полноценную инструкцию по настройке и генерации PDF-версии документации.

**Файл:** `ru/guides/generate-pdf.md`

## Структура новой страницы

1.  **Введение**
    *   Краткое описание возможности генерации PDF.
    *   Для кого это полезно (техписы, пользователи, open source).

2.  **Подготовка к работе**
    *   Требования: установленный `@diplodoc/cli`.

3.  **Настройка конфигурации (.yfm)**
    *   Включение генерации PDF (`enabled: true`).
    *   Пример конфигурации.

4.  **Запуск сборки**
    *   Команда сборки с флагом `--output-format=pdf` (или `--pdf`, уточню по контексту, обычно это формат вывода).
    *   Пояснение процесса: генерация single-page HTML -> конвертация в PDF.

5.  **Настройка структуры PDF**
    *   **Титульная страница:** Использование `startPages` в `toc.yaml`.
        *   Примечание про отсутствие локализации для этих страниц (ссылка на раздел Локализация).
        *   Отладка титульных страниц (`--pdf-debug` - *проверю, существует ли такой флаг, если нет - опишу как "просмотр HTML версии"*).
    *   **Оглавление:**
        *   Автоматическая генерация.
        *   Скрытие разделов из PDF (ссылка на `hidden` в `toc.yaml`).

6.  **Настройка внешнего вида**
    *   Использование CSS.
    *   Предупреждение об очистке (sanitization) инлайн-стилей.

7.  **Частые вопросы (FAQ)**
    *   Использование `presets` для условного контента (только для PDF).

## Изменения по сравнению с тезисами

*   Замена жаргонизмов ("санитайзится" -> "удаляются").
*   Добавление конкретных ссылок на разделы документации.
*   Форматирование команд и примеров кода.
*   Структурирование текста в виде пошаговой инструкции.
