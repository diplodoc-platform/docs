# Создание PDF из документации

<!-- Тезисы на основе записи встречи -->

Подходит для Open Source

Страница релевантна для техписов, которые хотят посмотреть, как будет выглядеть PDF локально

Установить PDF-генератор

Собрать документацию с флагом --pdf

Конфиг .yfm

```yaml
pdf:
  enabled: true
```

Папка output после сборки документации это папка input для PDF-генератора (вот тут не совсем понятно, как это выглядит на практике и как будет выглядеть консольная команда для локальной сборки?)

Для каждого файла оглавления toc.yaml формируется файл single-page.pdf. Порядок статей в PDF. определяется порядком статей в документации. Файлы с атрибутом hidden в PDF-сборку не попадают.

## Из чего состоит PDF

1. Титульная страница.

     Пользователь определяет статьи документации, названия которых попадут на титульную страницу. В оглавлении такие статьи учитываться не будут. Статьи определются в файле toc.yaml секцией pdf:

     ```yaml
    pdf:
	    startPages:
  	    - путь-до-статьи-1.md
  	    - путь-до-статьи-2.md
  	    - путь-до-статьи-3.md
     ```

    Статьи, указаные на титульно странице попадают в документацию перед оглавлением.

    Вызов сборки с параметром --pdf-debug генерирует страницы из startPages в HTML, чтобы можно было посмотреть, как они выглядят.

    Страницы, указанные в секции pds.startPages не поддерживаются yfm translate, то есть они не попадают в локализацию(перекрестная ссылка на раздел Локалзизация).

1. Оглавление.

    Оглавление генерируется автоматически на основе иерархии статей в файле tocy.yaml. Каждый пункт оглавления — ссылка на соответствующщую статью.
    Если пункт оглавления это группа статей, то это просто текст. Если хочется, чтобы такой пункт не попадал в оглавление, то можно настроить в конфиге соответствующий yfm-параметр (какой параметр? Нужно дать перекрестную ссылку).

1. Контент документации.

    Есть поддержка Page Constructor, работают перекрестные ссылки, корректно отображаются картинки.

## Стилизация PDF (предложить другой вариант заголовка)

Оформление PDF версии возможно с помощью кастомных [CSS-стилей](../style/css-js.md). Стили, которые использованы непосредственно в MD-файлах исходного кода документации будут санитайзится(заменить жаргонное слово)

## FAQ

Если хочется отобразить в PDF то, что не показано в html-версии, то придется использовать костыль через presets.yaml. То есть задать переменную для фильтрации содержимого только в PDF
